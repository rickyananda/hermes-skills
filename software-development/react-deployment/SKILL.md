# React + GitHub Pages Deployment

Deploy React apps (CRA) to GitHub Pages using the `gh-pages` npm package.

## Quick Steps

```bash
# 1. Create React app
npx create-react-app my-app
cd my-app

# 2. Install gh-pages
npm install gh-pages

# 3. Update package.json
```

### package.json (required fields)

```json
{
  "homepage": "https://USERNAME.github.io/REPO-NAME",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d build"
  }
}
```

### 4. Create GitHub repo & deploy

```bash
# Create repo via API
curl -s -X POST https://api.github.com/user/repos \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"REPO-NAME","public":true}'

# Init git & deploy
git init && git add -A && git commit -m "Initial commit"
git remote add origin https://USERNAME:TOKEN@github.com/USERNAME/REPO-NAME.git
npm run deploy
```

## ⚠️ CRITICAL PITFALLS

### 1. React 19 uses `createRoot` (NOT `ReactDOM.render`)

**This causes a BLANK WHITE PAGE with no error message!**

React 19 removed `ReactDOM.render`. Must use `createRoot`:

```js
// ❌ WRONG (React 18 and earlier)
import ReactDOM from 'react-dom';
ReactDOM.render(<App />, document.getElementById('root'));

// ✅ CORRECT (React 19+)
import { createRoot } from 'react-dom/client';
const root = createRoot(document.getElementById('root'));
root.render(<App />);
```

### 2. ESLint config errors on build

If you see `Failed to load config "airbnb"` or similar:

```bash
# Option A: Remove .eslintrc.js
rm .eslintrc.js

# Option B: Disable via env var
DISABLE_ESLINT_PLUGIN=true npm run deploy
```

### 3. Browser cache after deployment

After deploying updates, the old JS/CSS may be cached:
- Add `?v=2` query param to URL to bust cache
- Or use Ctrl+Shift+R (hard refresh)
- Wait 10-30 seconds for GitHub Pages CDN to propagate

### 4. Token authentication

Git remote URL format:
```
https://USERNAME:ghp_XXXX@github.com/USERNAME/REPO-NAME.git
```

For `gh-pages` package, it uses the git remote URL automatically.

**Token Management Pattern:**
- Store token in environment variable or config file
- When creating repo via API: `curl -H "Authorization: token $TOKEN" ...`
- When setting git remote: `git remote add origin https://USER:TOKEN@github.com/...`
- If token expired, user must provide new one — don't try to find old tokens in masked credentials

## Deployment URL

After `npm run deploy`, site is live at:
```
https://USERNAME.github.io/REPO-NAME/
```

GitHub Pages needs ~30s to update after each deploy.

## File Structure

```
my-app/
├── package.json     # Must have "homepage" field
├── public/
│   └── index.html
├── src/
│   ├── index.js     # Must use createRoot (React 19)
│   ├── App.js
│   └── App.css
└── build/           # Generated by npm run build
```
